## Part 1: 捐赠平台与方式对比

### 1.1 GitHub Sponsors

| 项目 | 详情 |
|------|------|
| **平台费用** | **0%**（个人赞助者）；最高 6%（组织赞助者） |
| **支付处理费** | Stripe 处理费（约 2.9% + $0.30） |
| **税务处理** | 美国纳税人需提交 W-9；年收入超 $600 发送 1099-NEC |
| **支付方式** | Stripe 直接打款到银行账户 |
| **设置难度** | 简单，直接在 GitHub 个人资料中设置 |
| **优势** | 与代码仓库深度集成；零平台费；开发者信任度高 |
| **劣势** | 仅限 GitHub 用户赞助；未来可能收取费用 |

**关键点：** GitHub Sponsors 是开源项目最推荐的平台。零平台费意味着你几乎拿到全部金额（仅扣除 Stripe 支付处理费）。GitHub 会为美国纳税人发送 1099-NEC 表格（年收入 >= $600 时）。从 2026 年起，1099-NEC 的申报门槛提升至 $2,000。

### 1.2 Open Collective / Open Source Collective

| 项目 | 详情 |
|------|------|
| **平台费用** | 0%（平台本身），但 Fiscal Host 收取费用 |
| **Fiscal Host 费** | Open Source Collective 收取 **10%** 的入账费用 |
| **税务处理** | OSC 负责税务申报、审计等合规事务 |
| **透明度** | 所有收支完全公开可见 |
| **法律实体** | OSC 是 501(c)(6) 非营利组织（非慈善机构） |
| **优势** | 完全透明；无需自己处理税务；适合团队项目 |
| **劣势** | 10% 费用较高；捐赠者的捐款**不能**抵税 |

**关键点：** Open Collective 适合团队项目或需要高透明度的场景。对于个人开发者的小型项目，10% 的费用偏高。

### 1.3 Ko-fi

| 项目 | 详情 |
|------|------|
| **平台费用** | 一次性捐赠 **0%**；会员订阅 **5%** |
| **Ko-fi Gold** | $29/月，解锁高级功能，无平台费 |
| **支付处理费** | Stripe/PayPal 处理费（约 2.9% + $0.30） |
| **税务处理** | 不提供税务表格，需自行报税 |
| **优势** | 一次性捐赠零平台费；界面友好 |
| **劣势** | 品牌辨识度不如 GitHub Sponsors |

### 1.4 Buy Me a Coffee

| 项目 | 详情 |
|------|------|
| **平台费用** | **5%**（所有交易） |
| **支付处理费** | 另加支付处理费 |
| **税务处理** | 不提供税务表格，需自行报税 |
| **优势** | 界面简洁友好；品牌知名度高 |
| **劣势** | 5% 一刀切费率；比 Ko-fi 贵（Ko-fi 一次性捐赠 0%） |

### 1.5 Patreon

| 项目 | 详情 |
|------|------|
| **平台费用** | **10%**（2025 年 8 月起新用户统一费率） |
| **支付处理费** | 约 2.9% + $0.30（另计） |
| **税务处理** | 发送 1099-K（达到门槛时） |
| **优势** | 强大的会员管理功能；适合内容创作者 |
| **劣势** | 10% 费率最高；更适合内容创作者而非开源项目 |

### 1.6 PayPal Donate Button

| 项目 | 详情 |
|------|------|
| **费用（个人/非慈善）** | **2.89% + $0.49**（每笔交易） |
| **费用（501c3 非营利）** | 1.99% + $0.49（折扣费率） |
| **国际交易附加费** | +1.50% |
| **优势** | 使用广泛；国际支持好 |
| **劣势** | 费率较高；$0.49 固定费对小额不友好 |

### 1.7 Stripe 直接支付

| 项目 | 详情 |
|------|------|
| **标准费率** | **2.9% + $0.30**（国内卡） |
| **国际卡** | 3.1% + $0.30 + 1.5% 跨境费 |
| **非营利折扣** | 2.2% + $0.30（需 501c3 资质） |
| **优势** | 灵活性最高；可完全自定义 |
| **劣势** | 需要自建支付页面；技术门槛最高 |

### 1.8 加密货币捐赠

| 项目 | 详情 |
|------|------|
| **费用** | 仅网络 Gas 费（通常很低） |
| **税务处理** | 收到加密货币视为收入，按市价计税 |
| **优势** | 匿名性；低费率；全球无障碍 |
| **劣势** | 用户群小；税务处理复杂；价格波动 |

| **Ko-fi（一次性）** | $0 | ~$29.50 | **~$470.50** | **94.1%** |
| **Buy Me a Coffee** | $25 | ~$29.50 | ~$445.50 | 89.1% |

## Part 2: 个人账户 vs 公司账户

### 2.1 捐赠收入到个人账户

**税务定性：**
- 开源项目的捐赠在 IRS 看来**不是**"礼物（gift）"——因为捐赠者通常期望你持续维护项目
- IRS 将此视为 **自雇收入（self-employment income）**
- 需要在 Schedule C 上报告
- 需缴纳 **自雇税（15.3%）** + **联邦所得税**

**报告要求：**
- GitHub Sponsors：年收入 >= $600（2025 年），或 >= $2,000（2026 年起）发送 1099-NEC
- 1099-K 门槛：$20,000 且 200 笔交易（One Big Beautiful Bill Act 恢复了原有高门槛）
- **即使不收到 1099 表格，你仍有义务报告所有收入**

**优点：**
- 设置简单，无额外成本
- 无需公司注册或维护

**缺点：**
- 无责任保护（liability protection）
- 个人资产面临风险
- 不够专业

### 2.2 捐赠收入到公司账户

**税务定性：**
- 作为 LLC 的营业收入（business income）
- 单成员 LLC 默认为"pass-through"实体，税务处理与个人相同
- 但提供了**法律责任隔离**

**优点：**
- **责任保护：** 公司债务不会影响个人资产
- **专业形象：** 商业银行账户和正式实体
- **费用抵扣更清晰：** 公司支出更容易作为业务费用扣除
- **未来灵活性：** 可以选择 S-Corp 税务选举来节省自雇税（收入较高时）

**缺点：**
- 需要维护公司合规（年报、州税等）
- 加州 LLC 每年需缴 **$800 最低特许税**（Franchise Tax）
- 需要独立记账

### 2.3 建议：使用公司账户

**推荐方案：通过公司（LLC）的银行账户接收捐赠。**

理由：
1. 你已经注册了公司，边际成本为零
2. 责任保护是最重要的考量
3. 将个人财务与项目收入分开是最佳实践
4. 所有业务费用（Apple Developer 账户、API 费用、设备等）可以清晰地作为业务支出扣除
5. 如果未来收入增长，可以选择 S-Corp 税务选举来节省自雇税

**是否需要单独开银行账户？**
- 如果你的公司已有银行账户，直接使用该账户即可
- **不要**将捐赠收入与个人银行账户混合
- 公司银行账户 + 记账软件（如 Wave 免费版）是最佳组合

## Part 3: 美国税务处理

### 3.1 开源捐赠的税务性质

**核心概念：** 开源项目收到的捐赠**不是**慈善捐赠（charitable donation），除非你的组织是 IRS 认可的 501(c)(3)。

- 对捐赠者：他们的捐赠**不能**用于个人所得税抵扣
- 对你：这是**应税收入**，视为自雇收入或营业收入

### 3.2 联邦税负

#### 自雇税（Self-Employment Tax）

| 组成部分 | 税率 | 说明 |
|---------|------|------|
| Social Security | 12.4% | 适用于前 $184,500 收入（2026 年上限） |
| Medicare | 2.9% | 无收入上限 |
| **合计** | **15.3%** | 基于净自雇收入的 92.35% |
| 额外 Medicare | +0.9% | 收入超过 $200,000 时（单身） |

**重要：** 你可以在个人所得税中扣除自雇税的 50%（Schedule 1 的 adjustment）。

#### 联邦所得税（2026 年税率）

| 应税收入区间（单身） | 税率 |
|---------------------|------|
| $0 - $11,925 | 10% |
| $11,926 - $48,475 | 12% |
| $48,476 - $103,350 | 22% |
| $103,351 - $197,300 | 24% |
| $197,301 - $250,525 | 32% |
| $250,526 - $626,350 | 35% |
| $626,351+ | 37% |

#### QBI 扣除（Qualified Business Income Deduction）

- 2026 年起，QBI 扣除比例提升至 **23%**（One Big Beautiful Bill Act 永久化并提升）
- 如果 QBI >= $1,000，最低扣除额为 $400
- 这是一项对 pass-through 实体（包括 LLC）的显著税务优惠

### 3.3 可抵扣的业务费用

以下支出可以作为业务费用（business expenses）在 Schedule C 上扣除，从而降低应税收入：

| 费用类别 | 示例 | 估计年费用 |
|---------|------|-----------|
| Apple Developer 账户 | $99/年 | $99 |
| OpenAI API 费用 | 开发测试用量 | 变动 |
| 云服务/服务器 | AWS、域名等 | 变动 |
| 开发设备 | Mac、显示器等（可折旧或 Section 179 一次性扣除） | 变动 |
| 软件订阅 | IDE、设计工具等 | 变动 |
| 互联网费用 | 家庭办公使用比例 | 部分可扣 |
| 家庭办公室 | 专用工作空间的面积比例 | 部分可扣 |
| 专业服务 | CPA、律师费用 | 变动 |
| 健康保险 | 自雇者 100% 可扣 | 变动 |

### 3.4 预估季度税

如果你预计年度欠税 >= $1,000，需要缴纳预估季度税（Estimated Quarterly Tax）：

| 截止日期 | 覆盖期间 |
|---------|---------|
| 4 月 15 日 | 1月1日 - 3月31日 |
| 6 月 15 日 | 4月1日 - 5月31日 |
| 9 月 15 日 | 6月1日 - 8月31日 |
| 1 月 15 日（次年） | 9月1日 - 12月31日 |

**提示：** 使用 IRS Form 1040-ES 计算预估税。如果不按时缴纳，可能面临罚款。

### 3.5 1099 表格的申报门槛（2026 年最新）

| 表格类型 | 门槛 | 说明 |
|---------|------|------|
| **1099-NEC** | $2,000（2026 年起） | GitHub Sponsors 等平台发送 |
| **1099-K** | $20,000 + 200 笔交易 | PayPal、Stripe 等第三方支付 |

> **One Big Beautiful Bill Act** 恢复了 1099-K 的 $20,000/200 笔交易高门槛，替代了之前计划的 $600 低门槛。这对小额捐赠收入者是好消息——但**即使不收到 1099，你仍需报告所有收入**。

### 3.6 加州（California）特殊考量

| 项目 | 详情 |
|------|------|
| **州所得税** | 1% - 13.3%（累进税率，最高档全美最高） |
| **LLC 最低特许税** | $800/年（首年豁免） |
| **LLC 额外费用** | 总收入 $250,000 - $499,999 -> $900；$500,000 - $999,999 -> $2,500 |
| **预估州税** | 也需要按季度缴纳 |
| **SDI** | 加州对自雇收入不收 SDI（State Disability Insurance），除非自愿选择 |

**加州 LLC 年成本基线：$800**（即使收入为零也要交，除非是成立的第一年可豁免）。

### 3.7 记录保存要求

- 保留所有收入记录（平台仪表盘截图、银行对账单）
- 保留所有支出收据（至少 3-7 年）
- 使用记账软件（推荐 Wave 免费版或 QuickBooks Self-Employed）
- 将个人支出和业务支出完全分开

## Part 4: 法律实体结构选项

### 4.1 结构对比

| 特性 | 个体经营 (Sole Prop) | 单成员 LLC | S-Corp 选举的 LLC |
|-----|---------------------|-----------|------------------|
| **设立费用** | $0 | $70-$500（因州而异） | LLC 费用 + 选举申请 |
| **责任保护** | 无 | 有 | 有 |
| **税务** | Schedule C | Schedule C（默认） | 工资 + 分红 |
| **自雇税** | 全部净收入 | 全部净收入 | 仅工资部分 |
| **合规负担** | 最低 | 年报 + 州税 | 工资处理 + 年报 |
| **适合收入水平** | <$5,000/年 | $5,000-$50,000/年 | >$50,000/年 |

### 4.2 针对你的建议

**你已经注册了公司（可能是 LLC），这是正确的选择。** 原因：

1. **责任保护：** 即使是免费开源软件，也可能面临诉讼风险（虽然概率很低）
2. **专业性：** 使用公司名义接收捐赠更专业
3. **费用扣除：** 公司结构让费用扣除更明确合理
4. **灵活性：** 未来收入增长时可以选举为 S-Corp 以节省自雇税

### 4.3 DBA（Doing Business As）

- 如果你的 LLC 名称不是 "WhisperUtil"，可以考虑申请 DBA
- DBA 允许你以项目名义开设银行账户和接收付款
- 费用通常 $10-$100（因州/县而异）
- 对于 GitHub Sponsors 等平台，通常不需要 DBA

### 4.4 接受捐赠是否改变商业注册要求？

- **不会。** 接受捐赠本身不需要额外的许可证或注册
- 但如果你在某些州提供"付费产品或服务"，可能需要销售税许可（donation 通常不适用）
- 某些州对 solicitation（主动募捐）有注册要求，但通常仅针对 501(c)(3) 慈善组织

## Part 5: 成功开源项目的最佳实践

### 5.1 知名项目的捐赠模式

| 项目 | 平台 | 模式 | 备注 |
|------|------|------|------|
| **Homebrew** | Open Collective | 社区透明捐赠 | 非营利项目，纯志愿者运营 |
| **Neovim** | GitHub Sponsors + Open Collective | 资金转入 OC 管理 | 也接受 Bitcoin |
| **curl** | 多渠道 | 企业赞助 + 基金 | 累计筹集超 $100K；曾获 Sovereign Tech Fund 资助 |
| **Vue.js** | GitHub Sponsors + Patreon + OpenCollective | 多平台组合 | 尤雨溪曾全职靠赞助 |

### 5.2 GitHub Sponsors 收入实际水平

| 层级 | 月收入范围 | 典型情况 |
|------|-----------|---------|
| 起步期 | $0-$50/月 | 大多数个人项目 |
| 成长期 | $50-$500/月 | 有稳定用户群的工具 |
| 成熟期 | $500-$3,000/月 | 知名项目维护者 |
| 顶级 | $3,000-$10,000+/月 | 极少数明星项目（如 Caleb Porzio 达到 $100K/年） |

**现实预期：** WhisperUtil 作为一个 macOS 语音转文字工具，定位是垂直领域的实用工具。初期合理预期大约在 $0-$100/月。如果工具获得大量用户和好评，可能达到 $100-$500/月。

### 5.3 有效的捐赠层级设计

| 层级 | 金额 | 建议回报 |
|------|------|---------|
| 咖啡 | $3-$5/月 | 感谢列表、Sponsor 徽章 |
| 支持者 | $10/月 | 上述 + 优先回复 issue |
| 赞助者 | $25/月 | 上述 + README 鸣谢 |
| 企业 | $100+/月 | 上述 + Logo 展示 + 优先功能请求 |

### 5.4 如何得体地请求捐赠

**推荐做法：**
- 在 README 底部添加简短的 "Support this project" 部分
- 使用 GitHub 的 `FUNDING.yml` 文件（自动在仓库页面显示 "Sponsor" 按钮）
- 在 Release Notes 中偶尔提及赞助
- 应用内添加"关于"页面，包含赞助链接

**避免：**
- 在应用中弹窗要求捐赠
- 频繁发送邮件催促
- 在 issue 回复中要求捐赠
- 将基本功能锁定在付费层级后面

**FUNDING.yml 示例：**
```yaml
github: [your-username]
ko_fi: your-kofi-username
custom: ["https://your-website.com/sponsor"]
```

## Part 6: 实际操作建议

### 6.1 分步行动计划

#### 第一阶段：基础设置（1-2 天）

1. **设置 GitHub Sponsors**
   - 在 GitHub 个人账户中启用 Sponsors
   - 提交 W-9 表格（使用公司 EIN）
   - 连接公司银行账户到 Stripe
   - 设置 3-4 个捐赠层级

2. **创建 FUNDING.yml**
   - 在仓库根目录创建 `.github/FUNDING.yml`
   - 添加 GitHub Sponsors 链接

3. **更新 README**
   - 在底部添加简短的赞助部分

#### 第二阶段：补充渠道（可选，按需添加）

4. **设置 Ko-fi**（备选捐赠渠道）
   - 一次性捐赠零平台费
   - 适合不使用 GitHub 的用户

5. **在应用内添加赞助链接**
   - 在"关于"窗口中添加 GitHub Sponsors 链接

#### 第三阶段：财务管理

6. **设置记账**
   - 开通公司银行账户（如果还没有）
   - 使用免费记账软件（Wave）记录收支
   - 保留所有业务支出收据

7. **税务准备**
   - 每季度估算税负并按时缴纳预估税
   - 年底前咨询 CPA

### 6.2 推荐平台组合

| 优先级 | 平台 | 用途 |
|--------|------|------|
| **主力** | GitHub Sponsors | 主要捐赠渠道，零平台费 |
| **备选** | Ko-fi | 一次性捐赠，面向非 GitHub 用户 |

**不推荐（目前阶段）：**
- Patreon（10% 太贵，且不适合开源项目定位）
- Open Collective（10% 费用，适合团队项目而非个人）
- 加密货币（税务复杂，用户群极小）

### 6.3 账户设置建议

```
捐赠收入 -> GitHub Sponsors (Stripe) -> 公司银行账户
                                          |
                                     记账软件记录
                                          |
                                   季度预估税缴纳
```

- **使用公司银行账户**接收所有捐赠收入
- **不要**混用个人账户
- GitHub Sponsors 注册时使用公司 EIN（如果是 LLC）

### 6.4 不同收入水平的税负估算

以下估算假设：加州单身纳税人，无其他收入，标准扣除，LLC pass-through。

#### 场景 A：$100/月（$1,200/年）

| 税种 | 估算 |
|------|------|
| 联邦所得税 | ~$0（标准扣除覆盖） |
| 自雇税 | ~$170 |
| 加州所得税 | ~$0（标准扣除覆盖） |
| 加州 LLC 特许税 | $800 |
| **总税负** | **~$970** |
| **有效税率** | **~81%**（被 $800 固定成本压垮） |

> 注意：加州 $800 LLC 年费在低收入时占比极大。如果你的公司已经注册且需要缴纳这笔费用，那么捐赠收入只是利用了已有的基础设施。如果仅为接受捐赠而注册 LLC，在低收入阶段不合算。

#### 场景 B：$500/月（$6,000/年）

| 税种 | 估算 |
|------|------|
| 联邦所得税（扣除标准扣除后） | ~$0-$200 |
| 自雇税 | ~$848 |
| 加州所得税 | ~$0-$100 |
| 加州 LLC 特许税 | $800 |
| **总税负** | **~$1,748-$1,948** |
| **有效税率** | **~29-32%** |

#### 场景 C：$1,000/月（$12,000/年）

| 税种 | 估算 |
|------|------|
| 联邦所得税 | ~$400-$800 |
| 自雇税 | ~$1,697 |
| QBI 扣除节省 | ~-$200 |
| 加州所得税 | ~$200-$400 |
| 加州 LLC 特许税 | $800 |
| **总税负** | **~$2,897-$3,497** |
| **有效税率** | **~24-29%** |

> 注：以上为粗略估算。实际税负取决于你的总体收入、扣除项、申报状态等因素。如果你有全职工作收入，捐赠收入会叠加到更高的税率档位。请咨询 CPA 获取准确计算。

### 6.5 税务准备清单

- [ ] 获取公司 EIN（如果还没有）
- [ ] 开通公司银行账户
- [ ] 设置记账软件，分类记录所有收支
- [ ] 每季度计算并缴纳预估联邦税和州税
- [ ] 保留所有业务支出收据（数字化存储）
- [ ] 年底汇总收入和支出
- [ ] 收集所有 1099 表格
- [ ] 提交联邦 Schedule C + Schedule SE
- [ ] 提交加州 Form 568（LLC 申报表）+ Form 3522（LLC 特许税）
