# 2026-04-15 Development Log / 开发日志

### 1. Naming Cleanup / 命名整理

- Renamed `UserSettings` → `SettingsDefaults` to reflect its role as fallback defaults for SettingsStore, not a runtime settings container
- 将 `UserSettings` 重命名为 `SettingsDefaults`，更准确地反映其作为 SettingsStore 的默认回退值的职责，而不是运行时的设置容器

- Renamed auto-send "smart" mode → "delayed"; the name was aspirational and the mode simply delays sending by a timer
- 将自动发送的「smart」模式改名为「delayed」；原名暗示了并不存在的「智能」逻辑，实际只是用计时器延迟发送

- Updated enum case, properties, methods, log strings, and UserDefaults keys consistently
- 统一更新枚举 case、属性、方法、日志字符串和 UserDefaults key

### 2. Cloud Transcription Language Fix / 云端转录语言修复

- Fixed cloud transcription returning English when user speaks Chinese on English macOS
- 修复了在英文系统下说中文却被云端转录为英文的问题

- Root cause: when `whisperLanguage` was empty (auto-detect), the code derived a language from system locale, which resolved to "en" on English macOS and forced English transcription
- 根因：当 `whisperLanguage` 为空（自动检测）时，代码从系统 locale 推导语言，在英文 macOS 上解析为 "en"，导致强制英文转录

- Fix: empty language now omits the API parameter entirely, letting the model truly auto-detect
- 修复：空语言设置不再拼装 API 参数，让模型真正自动识别

### 3. Dynamic Chinese Script Conversion / 动态中文脚本转换

- Replaced hardcoded `enableTraditionalToSimplified = true` with dynamic behavior driven by the user's recognition language: `zh` → simplified, `zh-Hant` → traditional, other languages pass through unchanged
- 将硬编码的 `enableTraditionalToSimplified = true` 改为基于用户识别语言的动态行为：`zh` 转简体、`zh-Hant` 转繁体，其他语言不做转换

- Fixed a related bug: selecting `zh-Hant` previously sent the raw code to the Whisper API, which only accepts `zh`. All three services now normalize language codes via `LocaleManager.whisperCode()` before calling APIs
- 顺便修复：选择 `zh-Hant` 时会把原始 code 直接发给只接受 `zh` 的 Whisper API。三个 Service 现在都在调用 API 前通过 `LocaleManager.whisperCode()` 规范化语言 code

### 4. RecordingController Refactor / RecordingController 重构

- Moved text post-processing (tag filtering, Chinese script conversion) from `TextInputter` to `RecordingController` — `TextInputter` is now a pure I/O utility with no knowledge of Whisper or language settings
- 将文本后处理（标签过滤、繁简转换）从 `TextInputter` 迁移到 `RecordingController`，`TextInputter` 回归为纯 I/O 工具，不再耦合 Whisper 或语言设置

- Extracted `AutoSendManager` (delayed send logic) and `TranslationPipeline` (two-step translation flow) into a new `Core/` directory
- 将 `AutoSendManager`（延迟发送逻辑）和 `TranslationPipeline`（两步翻译流程）抽取为独立模块，放入新的 `Core/` 目录

- Moved `RecordingController` and `HotkeyManager` into `Core/` to keep the project root clean
- 将 `RecordingController` 和 `HotkeyManager` 也移入 `Core/`，保持项目根目录整洁

- Extracted `TextPostProcessor` (tag filtering, Chinese script conversion) to `Utilities/`
- 将 `TextPostProcessor`（标签过滤、中文脚本转换）抽取到 `Utilities/`
