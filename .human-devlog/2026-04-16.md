- `AudioRecorder` 从双模式（streaming + 非 streaming）折叠成单模式，删除 `onAudioChunk`、`streamingMode`、`convertToPCM16` 等分支

### 3. Text Cleanup Feature Removal / 删除 GPT 文本润色功能

- Deleted the gpt-4o-mini text cleanup pipeline (`TextCleanupService` + 4-case mode enum + prompt templates). Transcription output now goes straight to the active window without an optional GPT rewrite step
- 删除基于 gpt-4o-mini 的文本润色流程（`TextCleanupService` + 四档模式枚举 + prompt 模板）。转录结果现在直接写入活动窗口，不再经过可选的 GPT 改写步骤

- Reasoning: the feature added latency and API cost with limited value — `TextPostProcessor` (tag filtering + 繁简转换) already handles the deterministic parts
- 理由：该功能带来延迟和 API 开销，但收益有限——`TextPostProcessor`（标签过滤 + 繁简转换）已经处理了确定性的后处理

### 5. Coding Principles in CLAUDE.md / 在 CLAUDE.md 写明编码原则

- Codified three project-wide principles in the root CLAUDE.md so subdirectory files can tighten but not contradict them:
  1. **Log generously** at decision points and boundaries
  2. **Defensive, but not paranoid** — validate at boundaries, never catch-and-silently-ignore
  3. **Good craftsmanship, not over-engineering** — YAGNI, no speculative abstractions
- 在根 CLAUDE.md 里明确三条全项目适用的编码原则，子目录文件可以收紧但不能反向冲突：
  1. **日志多打**——决策点和边界处主动打日志
  2. **防御但不偏执**——在边界处校验，绝不静默吞错
  3. **追求工艺但不过度设计**——YAGNI，不造推测性的抽象

- These crystallize patterns that were already showing up in recent work (language fallback hardening, removing dead `?? "en"` defaults)
- 这些其实是把最近实际已经在用的做法写下来（比如语言回退加固、移除无脑的 `?? "en"` 默认值）
