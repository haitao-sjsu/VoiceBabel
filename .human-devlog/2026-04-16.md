- Collapsed `AudioRecorder` from dual-mode (streaming + non-streaming) to single-mode; dropped `onAudioChunk`, `streamingMode`, and `convertToPCM16`
- `AudioRecorder` 从双模式（streaming + 非 streaming）折叠成单模式，删除 `onAudioChunk`、`streamingMode`、`convertToPCM16` 等分支

- 清理 ~24 条 Realtime 前缀的日志字符串和 4 条 UI 字符串，删掉 `ApiMode.realtime` 枚举

### 3. Text Cleanup Feature Removal / 删除 GPT 文本润色功能

- Deleted the gpt-4o-mini text cleanup pipeline (`TextCleanupService` + 4-case mode enum + prompt templates). Transcription output now goes straight to the active window without an optional GPT rewrite step
- 删除基于 gpt-4o-mini 的文本润色流程（`TextCleanupService` + 四档模式枚举 + prompt 模板）。转录结果现在直接写入活动窗口，不再经过可选的 GPT 改写步骤

- 理由：该功能带来延迟和 API 开销，但收益有限——`TextPostProcessor`（标签过滤 + 繁简转换）已经处理了确定性的后处理

### 5. Coding Principles in CLAUDE.md / 在 CLAUDE.md 写明编码原则

- Codified three project-wide principles in the root CLAUDE.md so subdirectory files can tighten but not contradict them:
  1. **Log generously** at decision points and boundaries
  2. **Defensive, but not paranoid** — validate at boundaries, never catch-and-silently-ignore
  3. **Good craftsmanship, not over-engineering** — YAGNI, no speculative abstractions
- 在根 CLAUDE.md 里明确三条全项目适用的编码原则，子目录文件可以收紧但不能反向冲突：
  1. **日志多打**——决策点和边界处主动打日志
  2. **防御但不偏执**——在边界处校验，绝不静默吞错
  3. **追求工艺但不过度设计**——YAGNI，不造推测性的抽象

- These crystallize patterns that were already showing up in recent work (language fallback hardening, removing dead `?? "en"` defaults)
- 这些其实是把最近实际已经在用的做法写下来（比如语言回退加固、移除无脑的 `?? "en"` 默认值）

### 13. Pipeline Refactor: Unified Input, Structured Output, Responsibility Realignment / Pipeline 重构：统一输入、结构化输出、职责归位

- **TranscriptionPipeline → TranscriptionManager**: now owns fallback state (`effectiveStartEngine`, `isInFallbackMode`) instead of delegating it to the controller. Accepts `[Float]` samples, returns `TranscriptionResult` with engine provenance (`engine`, `fallbackFrom`). Deleted `onFallbackEntered` callback — fallback is self-managed
- **TranscriptionPipeline → TranscriptionManager**：现在自管 fallback 状态（`effectiveStartEngine`、`isInFallbackMode`），不再委托给控制器。接受 `[Float]` 采样，返回带引擎来源信息的 `TranscriptionResult`（`engine`、`fallbackFrom`）。删除 `onFallbackEntered` 回调——fallback 完全内部处理

- **TranslationPipeline → TranslationManager**: simplified to pure text→text translation (Step 2 only). Removed `transcriptionPipeline` and `textInputter` dependencies. Returns `TranslationResult` with engine provenance. Two-step orchestration (transcribe then translate) moved to AppController
- **TranslationPipeline → TranslationManager**：简化为纯文本→文本翻译（只负责第二步）。移除 `transcriptionPipeline` 和 `textInputter` 依赖。返回带引擎来源信息的 `TranslationResult`。两步走编排（先转录再翻译）上移到 AppController

- **RecordingController → AppController**: merged three separate stop paths (`stopCloudRecording` / `stopLocalRecording` / `stopTranslationRecording`) into one `stopAndDispatch()`. `currentApiMode` and `isInFallbackMode` are now computed properties delegating to TranscriptionManager. New `startTranslation()` orchestrates the two-step flow
- **RecordingController → AppController**：三条独立的 stop 路径（`stopCloudRecording` / `stopLocalRecording` / `stopTranslationRecording`）合并为一个 `stopAndDispatch()`。`currentApiMode` 和 `isInFallbackMode` 改为计算属性，代理到 TranscriptionManager。新增 `startTranslation()` 编排两步走流程

- **CloudOpenAIService**: `transcribe()` now accepts `[Float]` samples and encodes internally via AudioEncoder (M4A/WAV). Old `Data`-based method removed. Added `.encodingFailed` error case (non-recoverable, won't trigger engine fallback)
- **CloudOpenAIService**：`transcribe()` 现在接受 `[Float]` 采样，内部通过 AudioEncoder 编码（M4A/WAV）。删除旧的基于 `Data` 的方法。新增 `.encodingFailed` 错误类型（不可恢复，不触发引擎回退）

- **AudioRecorder**: `stopRecording()` returns `[Float]?` directly. Deleted `RecordingResult`, `AudioFormat`, `getAudioSamples()`. `averageRMS` became a static utility method
- **AudioRecorder**：`stopRecording()` 直接返回 `[Float]?`。删除 `RecordingResult`、`AudioFormat`、`getAudioSamples()`。`averageRMS` 变为静态工具方法
